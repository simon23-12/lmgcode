## Architektur

### Virtual File System (vfs)
Alle Dateien leben nur im Browser-Memory (kein localStorage, kein Backend):
```js
vfs[path] = { content, language, model }
// model = Monaco ITextModel вҖ” wird einmal erstellt, nie neu erstellt beim Tab-Wechsel
```

### Kontext-Strategie
Nur **geГ¶ffnete Tabs** werden als Kontext ans Modell geschickt вҖ” nicht alle Dateien im Baum. Ein Badge im Chat-Header zeigt wie viele Dateien im Kontext sind. Max. 60.000 Zeichen pro Datei (dann gekГјrzt).

### Prompt-Struktur
```
[System-Instruktion]
[Projektdateien вҖ” nur offene Tabs]
[Aktuell sichtbare Datei]
[GesprГӨchsverlauf вҖ” letzte 10 Nachrichten]
[Aktuelle Frage]
```

### Serverless Function (`api/chat.js`)
- EmpfГӨngt `{ prompt: string, model: 'qwen' | 'step' | 'nemotron' | 'gemma', stream?: boolean }` вҖ” Prompt komplett im Frontend assembliert
- Probiert **genau ein Modell** pro Request вҖ” kein interner Fallback-Loop
- **Streaming-Modus** (`stream: true`): antwortet mit `Content-Type: text/event-stream` (SSE), SSE-Response direkt an Client gepipet
- **Non-Streaming-Modus**: antwortet mit `{ text, promptTokens, completionTokens }` oder `{ error, retryable }`
- Input-Limit: 500.000 Zeichen
- 57s AbortController-Timeout fГјr den initialen HTTP-Connect (Streaming-Funktionen: nur bis Headers empfangen)

### Fallback-Strategie (Frontend)
Fallback-Logik liegt im **Frontend** (`FALLBACK_CHAINS` in `index.html`), nicht im Backend. Jeder Retry ist ein neuer HTTP-Request вҶ’ neue Vercel-Instanz вҶ’ frische 300 Sekunden.

| GewГӨhltes Modell | Kette |
|---|---|
| Gemini Flash Lite (Standard) | Gemini Flash Lite вҶ’ Gemma вҶ’ Qwen вҶ’ Step вҶ’ Nemotron |
| Llama | Llama вҶ’ Gemini Flash Lite вҶ’ Qwen вҶ’ Step вҶ’ Nemotron вҶ’ Gemma |
| Qwen | Qwen вҶ’ Gemini Flash Lite вҶ’ Step вҶ’ Nemotron вҶ’ Gemma |
| Step | Step вҶ’ Nemotron вҶ’ Qwen вҶ’ Gemma |
| Nemotron | Nemotron вҶ’ Step вҶ’ Qwen вҶ’ Gemma |
| Gemma | Gemma вҶ’ Gemini Flash Lite вҶ’ Qwen вҶ’ Step вҶ’ Nemotron |
| Kimi | Kimi вҶ’ Step вҶ’ Gemini Flash Lite вҶ’ Qwen |

Die Fallback-Kette gilt fГјr **beide Modi** (streaming und non-streaming):
- Retryable-Fehler (429, 503, Timeout, Stream ohne `[DONE]` und ohne Content) вҶ’ nГӨchste Stufe
- Fatale Fehler (Auth, bad request) вҶ’ sofort stoppen

**Regel beim HinzufГјgen neuer Modelle:** Kette in `FALLBACK_CHAINS` in `index.html` ergГӨnzen + `MODEL_MAP` in `api/chat.js` ergГӨnzen.

### Multi-Agent-Modus
Mehrere unabhГӨngige Chat-Agenten kГ¶nnen gleichzeitig aktiv sein. Jeder Agent hat seinen eigenen State:

```js
agents: Map<id, {
  id, chatHistory, learnMode, isTyping, sessionTokens, model,
  srStreamBase, srStreamApplied, messagesEl  // eigener DOM-Pane
}>
```

- **Plus-Button** im Chat-Header spawnt einen neuen Agenten
- **Agent-Tabs** erscheinen automatisch ab 2 Agenten (Г—-Button schlieГҹt einzelne Agenten, letzter kann nicht geschlossen werden)
- Beim Tab-Wechsel (`switchAgent`) wird Modell-Dropdown und Lernmodus-Toggle auf den jeweiligen Agenten umgeschaltet
- Jeder Agent fГјhrt seinen eigenen Fallback-Chain-Durchlauf durch вҖ” parallel mГ¶glich
- `sendMessage()` und `buildPrompt()` nehmen das `agent`-Objekt als Parameter вҖ” kein globaler State mehr fГјr Chat
- `activePane()` liefert den DOM-Pane des aktiven Agenten fГјr `sysMsg`/`sysMsgHtml`

### Streaming-Architektur (`fetchStream` in `index.html`)
Live-Modus ist **immer aktiv** (`liveMode = true`). Code erscheint token-by-token im Monaco-Editor.

- `fetchStream(prompt, modelKey, onStreamStart, onSRDetected, agent)` вҖ” versucht genau ein Modell
  - SchlГӨgt fehl **vor** Stream-Start (429, Timeout auf Connect): gibt `{ error, retryable }` zurГјck, `onStreamStart` wird **nicht** aufgerufen вҶ’ Typing-Indikator bleibt, nГӨchstes Modell wird versucht
  - Stream startet erfolgreich: ruft `onStreamStart()` auf (Typing-Indikator weg), streamt Chunks
  - **Zweiphasiges Timeout-Modell:**
    - **Phase 1 вҖ” Connect:** 57s AbortController bis HTTP-Response-Headers (lГ¶st bei Streaming fast sofort auf, da Backend `res.flushHeaders()` sofort sendet)
    - **Phase 2 вҖ” First-Token:** 60s Timer startet nach `onStreamStart()`. Kommt kein Token вҶ’ `controller.abort()` вҶ’ `{ error, retryable: true }` вҶ’ Fallback. Kommt erster Token вҶ’ Timer gelГ¶scht вҶ’ Stream lГӨuft bis zu Vercels 300s Hard-Limit
  - Monaco-Update via `requestAnimationFrame` (throttled) вҖ” `extractStreamingCode()` parst offene/geschlossene CodeblГ¶cke
  - Stream endet mit `[DONE]` вҶ’ Erfolg: `{ text }`
  - Stream endet abrupt ohne Content вҶ’ `{ error, retryable: true }` вҶ’ nГӨchstes Modell
  - Stream endet abrupt mit partiellem Content вҶ’ gibt partial Text zurГјck (wird als Erfolg behandelt)
- Chat-Panel zeigt wГӨhrend des Streamings **nichts** вҖ” erst nach Abschluss erscheint Quip oder vollstГӨndige Antwort

### OpenRouter-Integration
- Kein SDK вҖ” nativer `fetch` gegen `https://openrouter.ai/api/v1/chat/completions` (OpenAI-kompatibel)
- Non-Streaming: `tryOpenRouter(prompt, orModel)` gibt Text zurГјck oder wirft
- Streaming: `streamOpenRouter(prompt, orModel, res)` pipet SSE-Body direkt via `res.write()`
- OpenRouter-Modelle: `qwen/qwen3.6-plus:free`, `stepfun/step-3.5-flash:free`, `nvidia/nemotron-3-super-120b-a12b:free`
- Google AI Studio: `gemma-4-31b-it` via `@google/generative-ai` SDK (rate-limited, kostenlos)
- Kriterien fГјr Free-Modelle: Programming-Ranking auf openrouter.ai/collections/free-models + Kontext вүҘ 128K (wegen groГҹer Prompts)
- $1-Spending-Limit auf dem OpenRouter-Key als Sicherheitsnetz; alle `:free`-Modelle kosten $0

## Frugal Coding Rules

- Single-file Frontend вҖ” kein Build-Step, kein Bundler
- Kein Framework, kein npm im Root
- Kein Datenbankbedarf (State lebt im Browser)
- Nur geГ¶ffnete Tabs als Kontext (nicht alle Dateien)

## Wichtige Pitfalls (Monaco)

- **CSP:** `unsafe-eval` + `blob:` + `worker-src blob:` in vercel.json zwingend вҖ” sonst kein Syntax-Highlighting
- **Worker:** `window.MonacoEnvironment.getWorkerUrl` muss auf CDN-Worker zeigen (via data-URL)
- **Resize:** `editor.layout()` manuell aufrufen вҖ” Monaco reagiert nicht auf CSS-Г„nderungen
- **Models:** Pro Datei genau ein `ITextModel` вҖ” beim Tab-Wechsel nur `editor.setModel()`, nie `dispose()` + neu erstellen (sonst geht Undo-History verloren)

## System-Prompt Philosophie

Der Assistent **schreibt Code wenn gewГјnscht** вҖ” er ist kein Lehrer der nur Hinweise gibt. Er erklГӨrt kurz was der Code macht, hГӨlt ErklГӨrungen aber knapp. Antwortet in der Sprache des SchГјlers (DE/EN).
